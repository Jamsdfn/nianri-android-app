# 念日

念日是一款简洁的 Android 重要日倒计时应用，用来记录生日、纪念日、节日以及任何值得期待的日子。

它同时支持新历（公历）和农历倒计时，可以把重要日子固定在桌面小部件上，并在指定时间发送提醒。

## 主要功能

- 按新历或农历计算每年下一次日期
- 独立设置日期展示方式，切换展示不会改变倒计时基准
- 支持提前 14 天、7 天、3 天提醒，并固定提供当天提醒
- 每个日子可单独设置提醒时间，默认每天 `09:00`
- 支持置顶重要日子
- 支持通过单个配置文件跨设备迁移全部重要日子
- 提供方形和宽版桌面小部件
- 每个小部件可独立选择日子和日期展示方式
- 小部件每天本地时间 `00:00` 静默更新倒计时
- 数据保存在本机，应用不申请网络权限

## 怎么使用

### 创建重要日子

1. 打开念日，点击右下角的“新建重要日子”。
2. 输入名称，例如“妈妈生日”或“结婚纪念日”。
3. 选择倒计时基准：
   - 新历基准：每年按公历月日计算。
   - 农历基准：每年按农历月日计算。
4. 选择日期以及默认展示方式。
5. 设置提醒时间，并按需开启提前 14 天、7 天或 3 天提醒。
6. 如需优先显示，可开启“设为置顶日子”。
7. 保存后，首页会显示距离下一次日期还有多少天。

当天提醒固定开启，不需要额外勾选。遇到 2 月 29 日或农历三十等并非每年都存在的日期，应用会按对应年份进行调整，并在界面中说明。

### 查看和编辑

- 点击首页中的日子可查看新历、农历对应日期和提醒设置。
- 在详情页进入编辑，可以修改名称、倒计时基准、展示方式、提醒时间和置顶状态。
- 首页支持在新历与农历展示之间切换；该操作只改变日期写法，不改变倒计时的计算基准。

### 添加桌面小部件

1. 在手机桌面长按空白位置，进入“小部件”列表。
2. 找到“念日”，选择方形或宽版样式并添加到桌面。
3. 在配置页选择一个重要日子和日期展示方式。
4. 点击“保存小部件”。

不同小部件可以选择不同的日子。小部件会在本地时间每天 `00:00` 自动刷新，也会在编辑日子或重新打开应用时更新。

### 跨设备迁移

1. 在 A 设备首页点击“迁移”，切换到“导出”，选择“导出全部配置”。
2. 将生成的 `.nianri.json` 文件发送或复制到 B 设备。
3. 在 B 设备首页点击“迁移”，切换到“导入”，选择该配置文件。

导入会与 B 设备已有数据合并，不会删除原有纪念日。若名称完全相同，导入项会自动追加导入日期；如果 B 设备已有置顶纪念日，也会保留 B 设备原来的置顶。小部件绑定和系统权限属于设备状态，不会随配置文件迁移。

## 提醒与权限说明

为了按设定时间准时提醒，应用可能需要以下系统权限：

- 通知权限：用于显示重要日提醒。
- “闹钟和提醒”权限：用于在指定时间触发提醒，以及安排每天零点的小部件更新。

零点小部件更新使用的是应用内部定时回调，不是用户可见的系统闹钟，不会响铃、震动或发送通知。它也不会使用 `setAlarmClock()` 创建闹钟条目。

如果没有授予“闹钟和提醒”权限，应用不会崩溃，但提醒和精准零点刷新可能延后；打开应用时仍会主动刷新。若用户在系统设置中强制停止应用，Android 会暂停其后台任务，重新打开应用后会恢复安排。

## 构建项目

项目最低支持 Android 8.0（API 26）。使用 Android Studio 打开项目，或在项目根目录执行：

```bash
./gradlew assembleDebug
```

生成的调试安装包位于：

```text
app/build/outputs/apk/debug/app-debug.apk
```

运行 JVM 测试：

```bash
./gradlew testDebugUnitTest
```

连接模拟器或设备后运行 Android 测试：

```bash
./gradlew connectedDebugAndroidTest
```

### 正式发布构建

正式构建需要通过环境变量提供发布签名，密钥文件和密码不要提交到 Git：

```bash
export NIANRI_RELEASE_STORE_FILE="/absolute/path/to/nianri-release.jks"
export NIANRI_RELEASE_STORE_PASSWORD="your-store-password"
export NIANRI_RELEASE_KEY_ALIAS="nianri"
export NIANRI_RELEASE_KEY_PASSWORD="your-key-password"

./gradlew assembleRelease bundleRelease
```

签名 APK 和 AAB 分别生成在：

```text
app/build/outputs/apk/release/app-release.apk
app/build/outputs/bundle/release/app-release.aab
```
