# 小部件午夜静默刷新设计

## 目标

每天本地时间 00:00 自动重新计算并刷新全部已配置小部件，使倒计时在跨自然日后及时变化。刷新过程完全静默：不发通知、不播放声音、不震动、不亮屏，也不在系统中创建用户可见的闹钟。

## 方案选择

采用 `AlarmManager.setExactAndAllowWhileIdle()` 安排一次性的下一个本地午夜回调。该 API 只负责在指定时刻派发应用内部 `PendingIntent`；实现不使用 `setAlarmClock()`，也不调用通知、音频或振动 API。

不采用以下方案作为主路径：

- `updatePeriodMillis` 最快只能按系统周期刷新，无法保证 00:00，且持续周期更新会产生不必要的唤醒。
- WorkManager 周期或延迟任务受 Doze、后台限制和厂商省电策略影响，不能保证精确零点。现有 WorkManager 审计继续保留为延迟兜底。

## 组件边界

### 午夜调度器

新增独立的 `MidnightWidgetRefreshScheduler`，只负责：

- 根据注入的 `Clock` 和当前系统时区计算下一个本地自然日的开始时刻。
- 使用固定身份的显式 `PendingIntent` 安排 `RTC_WAKEUP` 精确闹钟。
- Android 12 及以上先检查 `canScheduleExactAlarms()`；权限不可用时安全跳过，不抛出导致应用崩溃的异常。
- 重复调用时以同一个 `PendingIntent` 替换既有安排，保持幂等。

午夜计算使用“本地日期加一天后 `atStartOfDay(zone)`”，而不是固定增加 24 小时，因此能够正确处理时区和夏令时变化。

### 午夜 Receiver

新增未导出的 `MidnightWidgetRefreshReceiver`，只接受应用自身显式 `PendingIntent`。收到回调后通过 `goAsync()`：

1. 调用现有 `ConfiguredWidgetUpdater.updateAll()` 刷新全部已配置小部件。
2. 无论刷新成功或部分失败，都尝试安排下一个本地午夜。
3. 始终结束 `PendingResult`，避免 Receiver 超时泄漏。

Receiver 不创建通知渠道、不调用 `NotificationManager`、不播放声音或振动。

### 生命周期接入

- `AppContainer` 提供单例午夜调度器。
- `NianriApplication.onCreate()` 在进程启动时幂等安排下一个午夜。
- `SystemChangeReceiver` 在开机、应用升级、系统时间变化、时区变化以及精确闹钟权限重新授予后，完成现有提醒和小部件恢复流程，再重新安排下一个午夜。
- 当前 `DATE_CHANGED` 广播监听可以保留为额外兼容路径，但不再依赖它保证午夜刷新。

## 权限与降级

应用已经因为用户可见的重要日提醒声明并请求 `SCHEDULE_EXACT_ALARM`，午夜刷新复用同一能力，不增加新的权限弹窗。

如果权限未授予或被撤销：

- 午夜调度器不安排精确闹钟，也不崩溃。
- 现有 24 小时 WorkManager 审计继续进行延迟刷新。
- 用户打开应用时的立即审计继续刷新小部件。
- 权限重新授予后，现有系统广播恢复路径会重新安排午夜闹钟。

## 错误处理

- 单个小部件刷新失败时，现有批量更新器继续尝试其余实例。
- 午夜刷新整体失败不能阻止下一天的闹钟续订。
- 安排闹钟失败不会产生通知、前台服务或无限重试；后续应用启动、系统变化和 WorkManager 审计仍可恢复。
- 设备被用户强制停止应用后，Android 会禁止后台组件；这种状态只能在用户再次打开应用后恢复，符合系统规则。

## 测试范围

- 纯时间计算：普通日期、接近午夜、非整点时区及夏令时边界都得到下一本地自然日 00:00。
- 调度器：使用 `RTC_WAKEUP`、`setExactAndAllowWhileIdle`、固定显式 `PendingIntent`；重复安排不产生多个身份；无权限时不安排且不抛异常。
- Receiver：成功时刷新并续订；刷新失败时仍续订；最终总是完成异步接收。
- 生命周期：应用启动和各系统恢复动作都会请求重新安排午夜刷新。
- Manifest：声明未导出的午夜 Receiver，不增加通知、振动、音频、网络或新的精确闹钟权限。
- 回归：现有提醒调度、WorkManager 审计、系统变化恢复和小部件交互测试继续通过。

## 不在本次范围内

- 用户自定义小部件刷新时间。
- 每分钟或每小时刷新。
- 显示任何午夜刷新通知、状态栏图标或系统闹钟条目。
- 在用户强制停止应用后绕过 Android 限制自动恢复。
