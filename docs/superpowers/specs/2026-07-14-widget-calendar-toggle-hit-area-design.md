# 小部件日期展示切换热区设计

## 背景与问题

念日的 2×1 与 2×2 小部件当前在内容容器上绑定“打开日子详情”，同时仅在右下角日期文字上绑定“切换新历/农历展示”。日期文字的 RemoteViews 命中范围接近文字本身；用户点击文字周边空白时，会落到外层容器并打开 App，造成切换不灵敏。

这里的切换仍然只改变展示历法，不改变日子的倒计时基准。

## 选定方案

将内容区拆成两个不重叠的点击区域，视觉样式和文字内容保持不变：

- 上方主要内容区：点击打开对应日子的 App 详情页。
- 整条底部日期栏：点击切换该小部件的新历/农历展示。

2×1 与 2×2 小部件采用相同交互规则。底部日期栏中的日期说明、留白以及切换文字都属于切换热区，避免依赖精确点击小字号文字。

## 实现边界

- 调整 `NianriWidgetSurface` 的布局层级和点击绑定，移除覆盖整个卡片的详情点击父容器。
- 为主要内容区和底部日期栏分别绑定独立 Action。
- 不修改日期换算、倒计时、数据库、提醒或小部件选择日子的逻辑。
- 不改变当前小部件尺寸、颜色、文案和信息密度。
- API 26 大字体下 2×1 第二行会按现有兼容规则隐藏；该状态没有可见日期栏，因此仍由整个可见内容打开 App，不新增不可见切换区域。

## 验证

- 增加布局/结构测试，证明底部日期栏拥有独立且横向铺满的切换容器，详情点击不再绑定于共同父容器。
- 保留并运行现有展示切换控制器测试，确认切换只更新目标小部件。
- 在 API 26、31、36、37.1 模拟器运行相关测试。
- 重新安装到小米 15 Pro，真机验证底部日期栏左侧、中央、右侧均能切换，上方内容稳定打开 App。
