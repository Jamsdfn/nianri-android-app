# 剪贴板与文本配置迁移设计

## 背景与目标

念日已经支持把全部纪念日保存为 `.nianri.json` 文件，并从文件合并导入。为了减少跨设备传文件的步骤，本次在保留文件迁移的基础上增加两种入口：

- 导出配置时可直接复制到系统剪贴板。
- 导入配置时可在多行输入框中粘贴 JSON 并直接导入。

剪贴板与文本入口只扩展传输方式，不改变现有 JSON 格式、字段、重名规则、置顶规则或事务语义。

## 用户界面

首页“迁移”入口与迁移面板顶部“导出 / 导入”Tab 保持不变。不增加第二层 Tab。

### 导出 Tab

继续显示将导出的纪念日数量和配置范围说明，并提供两个操作：

- 主按钮“保存配置到本机”：沿用现有 Storage Access Framework 创建文档流程。
- 次按钮“复制配置到剪贴板”：生成与文件导出完全相同的 UTF-8 JSON 文本，并写入系统剪贴板。

没有纪念日或正在处理时，两个导出按钮均禁用。

复制成功后显示“配置已复制到剪贴板”。面板同时显示简短隐私提示：“剪贴板配置包含纪念日名称，请注意隐私。”

### 导入 Tab

保留文件入口并扩展文本入口，顺序如下：

1. “选择配置文件”按钮，沿用现有 Storage Access Framework 打开文档流程。
2. “或粘贴配置”分隔说明。
3. 多行文本输入框，允许用户手动输入、编辑或使用系统长按粘贴。
4. “从剪贴板粘贴”次按钮。
5. “导入粘贴内容”主按钮。

只有用户点击“从剪贴板粘贴”时，应用才主动读取系统剪贴板。读取到文本时用该文本完整替换输入框当前内容。

输入框为空或正在处理时，“导入粘贴内容”禁用。正在处理时，文件选择、快捷粘贴和文本导入操作都禁用，避免并发操作。

文本导入失败时保留输入内容，方便用户检查或修改；导入成功后清空输入框。切换“导出 / 导入”Tab 或关闭再打开迁移面板时，未成功导入的文本继续由 ViewModel 保留，直到成功导入、用户手动编辑为空或应用进程结束。

## 数据与组件边界

### 格式复用

剪贴板复制内容必须与同一时刻的文件导出内容使用相同的 `TransferCodec` 和版本 `1` 格式。文本导入必须调用现有 `ConfigurationTransferService.importConfiguration`，因此继续获得：

- 完整结构与严格字段类型校验
- B 设备已有数据保留
- `原名称-yyyyMMdd导入` 与数字序号兜底
- B 设备置顶优先
- 单次 Room 事务写入
- 提交后提醒重建和小部件刷新

不新增“剪贴板专用格式”、Base64 包装、压缩、链接或云服务。

### `TransferViewModel`

`TransferUiState` 增加 `importText: String`。ViewModel 增加以下职责：

- 更新输入框文本。
- 通过现有导出服务生成 JSON 后，把文本交给调用方写入剪贴板。
- 为文件导出与剪贴板复制显示不同成功文案。
- 使用 `importText` 执行文本导入。
- 只在文本导入成功后清空 `importText`。
- 接收“剪贴板为空”和剪贴板读写失败结果并转成用户可见消息。

文件内容、剪贴板内容和手动输入内容最终都通过相同的服务接口进入 codec 与导入事务。

### Compose UI

`TransferSheet` 继续是纯状态渲染组件。它接收新增的文本变化、复制、粘贴和文本导入回调，不直接访问 Android 系统服务。

输入框使用 Material 3 多行 `OutlinedTextField`，显示“粘贴导出的念日配置”占位提示。输入不做实时解析；只有用户点击导入按钮时才执行完整校验，避免在输入过程中不断显示错误。

### Android 剪贴板接线

`NianriNavHost` 使用应用上下文取得 `ClipboardManager`：

- 复制时调用 `ClipData.newPlainText("念日配置", json)` 并设置为 primary clip。
- 快捷粘贴时读取 primary clip 第一项，通过 `coerceToText(context)` 转成字符串。
- 剪贴板没有条目或转换后为空白时，不覆盖当前输入框，并提示“剪贴板中没有可粘贴的配置”。

Android 类型不进入 codec、service 或 repository。

## 状态与消息

新增或调整以下用户可见消息：

- 剪贴板复制成功：“配置已复制到剪贴板”
- 剪贴板无文本：“剪贴板中没有可粘贴的配置”
- 剪贴板复制失败：“复制失败，请重试”
- 剪贴板读取失败：“读取剪贴板失败，请重试”

现有文件导出成功消息“已导出 N 个纪念日”保持不变。现有文本解析、版本、字段、数据库和刷新错误文案保持不变。

剪贴板复制与文本导入复用当前 `isProcessing`，同一时间只允许一个导出或导入操作。单纯编辑输入框不进入处理中状态。

## 测试策略

### ViewModel 单元测试

- 文件导出与剪贴板复制得到相同 JSON。
- 文件导出保留“已导出 N 个纪念日”，复制成功显示“配置已复制到剪贴板”。
- 更新文本输入后状态立即反映新内容。
- 文本导入使用当前输入内容。
- 文本导入成功后清空输入内容。
- 文本导入失败时保留输入内容并显示原有精确错误。
- 处理期间拒绝第二个文件、复制或文本导入操作。
- 剪贴板为空和读写失败映射到对应消息。

### Compose UI 测试

- 导出 Tab 同时显示“保存配置到本机”和“复制配置到剪贴板”。
- 两个导出按钮在无数据或处理中均禁用。
- 导入 Tab 同时显示文件入口、多行输入框、快捷粘贴与文本导入按钮。
- 输入变化回调收到完整文本。
- 空文本时禁用文本导入；非空文本时可以触发导入。
- 处理中禁用文件选择、快捷粘贴和文本导入。
- 隐私提示可见。

### Android 集成测试

- 复制操作把版本化 JSON 写入 primary clip。
- 快捷粘贴读取 primary clip 并填充输入框。
- 空剪贴板不覆盖输入框，并显示空剪贴板提示。
- 现有 `CREATE_DOCUMENT` 与 `OPEN_DOCUMENT` launcher 测试继续通过。

### 回归测试

- `TransferCodec`、`ImportPlanner`、原子 repository、service、提醒与小部件测试全部继续通过。
- merged manifest 不新增存储或网络权限。

## 验收标准

- 用户可以在“导出”Tab 选择保存文件或复制配置，两者内容格式一致。
- 用户可以在“导入”Tab 选择配置文件，或粘贴/编辑文本后导入。
- 文本导入与文件导入使用完全相同的数据校验和合并规则。
- 空剪贴板、非法文本和系统剪贴板失败均有明确反馈，不造成数据写入。
- 文本导入成功清空输入；失败保留输入。
- 原有文件迁移能力、权限边界和测试继续有效。
