# 念日 Android 应用图标设计

## 目标

为“念日”建立正式、可辨识且适合小米 HyperOS 桌面的应用图标。图标需要同时表达新历与农历，但不能暗示用户切换了倒计时基准；核心含义是两种日期写法共同指向同一个重要日。

## 已确认方向

采用 B1「暮光紫双历日签」。

- 外层背景：从浅暮光紫到深紫的柔和渐变，与 App 和小部件现有夜空紫视觉保持一致。
- 主体：位于中央安全区的白色圆角日签。
- 新历符号：日签左下的暖黄色太阳圆点。
- 农历符号：日签右下的紫色月牙。
- 日签顶部：两枚暖黄色装订环与一条浅紫分隔线。
- 不在图标内放“念日”文字，避免小尺寸下模糊，并让系统桌面标签负责展示应用名。

## 语义

太阳与月牙同处一张日签，表示新历与农历只是同一个重要日期的两种展示方式。它与产品规则一致：创建时选择的历法始终是倒计时基准，界面切换只改变展示日期。

## Android 资源

实现以下资源：

1. Adaptive icon background：纯色或近似渐变的暮光紫背景。
2. Adaptive icon foreground：白色日签、装订环、太阳和月牙，关键内容位于中央约 66% 安全区。
3. Android 13+ monochrome icon：用单色线框日签、太阳圆点和月牙构成，供系统主题图标着色。
4. Legacy launcher icon：为 API 26 及普通启动器提供完整合成图。
5. Round icon：与 adaptive icon 共用图形语义，圆形裁切不损失关键元素。

Manifest 配置 `android:icon` 与 `android:roundIcon`。资源名称统一使用 `ic_launcher`、`ic_launcher_round`、`ic_launcher_foreground`、`ic_launcher_background` 和 `ic_launcher_monochrome`。

## 视觉参数

- 主紫：`#7466E8`
- 浅紫：`#A093FF`
- 深紫：`#5147B8`
- 日签白：`#FAF9FF`
- 分隔浅紫：`#D9D4FF`
- 太阳与装订环：`#FFD482`

图形必须在 48dp 桌面图标尺寸仍能区分日签、太阳和月牙。避免细于约 2dp 的关键笔画，避免阴影承担结构信息。

## 兼容与验证

- API 26：验证 legacy/adaptive 图标能安装、启动器能读取，且没有系统默认占位图标。
- API 31、36、37.1：验证 adaptive icon 与圆形资源解析。
- Android 13+：验证 monochrome themed icon 存在且能被系统着色。
- 使用圆形、标准圆角方形与小米风格超椭圆遮罩检查安全区。
- 在浅色和深色壁纸上以桌面小尺寸检查辨识度。
- 构建后审计 merged manifest 的 `icon` 与 `roundIcon` 引用，并运行既有完整测试、Lint 与 APK 构建。

## 非目标

- 不改变 App 内页面或小部件的布局、配色和业务功能。
- 不设计启动动画、商店宣传图或新的应用名称。
- 不用文字、数字或具体日期作为图标主体。
