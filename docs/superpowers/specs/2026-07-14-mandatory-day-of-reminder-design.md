# 当天固定提醒与补发设计

## 目标

每个重要日子固定在当天上午 09:00 提醒。该提醒不可关闭；用户仍可分别开启或关闭提前 14、7、3 天提醒。

手机关机、系统延迟、App 被停止或通知权限暂时不可用时，只要在重要日子当天恢复且已经过 09:00，就补发当天提醒一次。次日不补发昨日提醒。

## 提醒规则

- 固定偏移 `0`：所有日子强制启用，触发时间为本地时区 09:00。
- 可选偏移 `14`、`7`、`3`：继续由用户独立配置。
- 当天 09:00 后新建一个恰好今天发生的日子，立即补发。
- 当天 09:00 后重新允许通知权限，App 恢复审计时立即补发。
- 农历或新历只影响每年发生日期的计算来源；提醒仍以换算后的本次公历日期触发。
- 非闰年或短农历月份提前一天的既有规则继续生效，提醒文案保留“本次提前 1 天”。

## 调度与补发

`AndroidReminderScheduler` 每次替换日子提醒时始终包含偏移 `0`，再合并用户选择的 14/7/3。若 09:00 尚未到，设置精确且允许空闲触发的闹钟；若本次发生日就是今天且已经达到 09:00，则立即派发同一个偏移 `0` Receiver，由通知服务和去重账本决定是否补发。这样保存日子、重建提醒和系统恢复都复用一条路径。

补发审计在以下入口运行：

- 系统开机完成；
- App 升级、日期变化、时间变化、时区变化、精确闹钟权限变化；
- App 启动及每次从系统设置返回前台时的一次性审计；
- 既有周期审计；
- 当天日子保存或提醒权限恢复后的下一次审计。

审计通过重建所有提醒触发上述即时派发路径。通知服务仍会重新校验本地日期，确保只补发今天的日子。

## 去重

使用 App 私有 SharedPreferences 保存每个日子最后成功发送的当天提醒公历日期，键由日子 ID 构成，值为 ISO 日期。进程内使用同一个互斥锁完成“检查并标记”，避免闹钟与审计并发重复。发送偏移 `0` 前检查账本；成功提交本次日期后才发送通知。若通知调用抛出异常，撤销本次标记以允许后续重试。

偏移 0 的通知使用稳定通知 ID，并启用 `setOnlyAlertOnce(true)`，作为系统通知层的第二道重复保护。14/7/3 不使用当天账本。

## 权限与界面

- 编辑页提醒区域增加只读文案“当天 09:00 · 固定开启”。
- 14/7/3 继续显示独立开关。
- 即使 14/7/3 全部关闭，权限状态也不能显示“不需要提醒”，因为当天提醒始终存在。
- Android 13+ 仍需用户授予通知权限；Android 12+ 若精确闹钟权限不可用，继续显示既有权限引导。产品内“强制开启”表示没有业务开关，不能绕过操作系统权限。

## 兼容与验证

- 不修改重要日子表和提醒掩码格式；偏移 0 不写入 `reminderMask`。
- 调度测试覆盖：可选提醒全关仍安排当天 09:00；可选开启时为 0/14/7/3 四个唯一闹钟。
- 通知测试覆盖：偏移 0 不依赖用户掩码；09:00 前不补发；09:00 后当天补发；同日重复审计不重复；次日不补发昨日。
- 恢复测试覆盖开机、App 启动、周期审计和失败重试。
- UI 测试覆盖固定提醒文案、14/7/3 独立开关与权限状态。
- 最终在 API 26、31、36、37.1 和小米 15 Pro 验证。
